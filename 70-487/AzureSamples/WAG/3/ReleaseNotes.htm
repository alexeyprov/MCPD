Release Notes for the Examples

System Requirements

 

  • Microsoft Windows 7 with Service Pack 1 or later (32 bit or 64 bit edition), or Windows Server 2008 R2 with Service Pack 1 or later
  • Microsoft Internet Information Server (IIS) 7.0
  • Microsoft .NET Framework version 4.0
  • Microsoft ASP.NET MVC Framework version 3
  • Microsoft Visual Studio 2010 Ultimate, Premium, or Professional edition with Service Pack 1 installed
  • Windows Azure SDK for .NET (includes the Visual Studio Tools for Windows Azure)
  • Microsoft SQL Server or SQL Server Express 2008
  • Windows Identity Foundation
  • Enterprise Library 5 (required assemblies are included in the source code download)
  • Windows Azure Platform Cmdlets (install the Windows Azure Platform Cmdlets as a PowerShell Snap-in, this is required for scripts that use the Azure Management API)
  • TreyResearch and TreyResearch.HeadOffice example databases (SQL scripts are included in the \Source\Setup\DependencyChecker\SqlScripts folder of the example)

To check for software prerequisites needed to use the examples, run CheckDependencies.cmd in the example scenarios folder using an administrator account. You must have .NET Framework version 4.0 installed to run the Dependency Checker.

 

For instructions on how to configure and run the example, see the Readme file.

 

Known Issues

 

Installing the pre-requisite Windows components

  • Ensure that you have installed all applicable updates for your computer from Windows Update before installing the pre-requisites listed above.
  • If you want to install the Windows Azure SDK for .NET on Windows Server 2008 R2, you must first ensure that the .NET Framework version 3.5.1 is installed on your computer.
  • If installation of the latest version of Windows Azure SDK for .NET fails when using the dependency checker, you can install it directly from the Microsoft download center.

 

Installing the databases manually

  • The Dependency Checker automatically installs the databases for the example into SQL Server Express or SQL Server, depending on which you have installed on your machine. If you have more than one version of either SQL Server Express or SQL Server installed, the Dependency Checker may install the databases into an instance of a version earlier than 2008. In this case you must manually install the databases using the SQL scripts provided using the following steps:
    • Execute the five scripts in the Setup\DependencyChecker\SqlScripts folder of the examples using SQL Server manager or the osql command line utility (for example, osql -E -S mycomputerName\SQLExpress -i create_objects_TreyResearch.sql) in the following order:
      • TreyResearch-DropDatabase.sql (if the databases already exist in your database server)
      • create_TreyResearch.sql
      • create_TreyResearch.HeadOffice.sql
      • create_objects_TreyResearch.sql
      • create_objects_TreyResearch.HeadOffice.sql
    • Grant access to the TreyResearch and TreyResearch.HeadOffice databases to your computer's NETWORK SERVICE account.

 

After Installing IIS

  • Make sure that the Load User Profile option for the ASP.NET v4.0 Application Pool is set to True in IIS:
    • Open Internet Information Services (IIS) Manager from your Start menu
    • Expand the left-hand list of Connections and select Application Pools
    • In the list of application pools, right-click ASP.NET v4.0 and click Advanced Settings
    • Find the Process Model section and ensure that the setting for Load User Profile is set to True
  • Make sure that port 80 and 443 are available. The Windows Azure Compute Emulator attempts to use the default ports for web applications, such as 80 (http) and 443 (https). If you run the example on a computer that has Internet Information Services (IIS) installed, or another application that uses these ports, the local Compute Environment will automatically use an alternative port. This will cause the application to fail when using the default configuration, which assumes the standard ports. There are three ways that you can resolve this:
    1. 1. Remove the existing sites from IIS and remove the existing bindings for port 80 and 443 in IIS Manager.
    2. 2. Remap the ports that IIS uses in IIS Manager so that ports 80 and 443 are available.
    3. 3. Change the ports used by the example. This will require you to edit the configuration of the example and your Windows Azure services.

 

Running the example

  • You must run the projects and solutions under an account that has Administrator privileges.
  • The example code is not designed to be reused as it stands in a production application.
  • The example uses several Windows Azure services (such as Access Control, Service Bus, and Caching) as well as Web Role, Worker Role, and SQL Azure instances. Your Azure account will be charged for the use of these services. Ensure that you stop or remove these after you finish running the example.
