Introduction

This sample demonstrates the different ways of authenticating an HTTP/REST client with Service Bus

Building the Sample

  • Install Windows Azure SDK 2.0 or later.
  • Create a Service Bus namespace.
  • In file Program.cs, replace the namespace name and key or credentials with your information.
  • Depending on the token provider you want to use, uncomment the appropriate secion in file Program.cs.
  • Build the sample. Open the solution in Visual Studio 2010 or
    later and press F6.
  • Press F5 to run the client.

Description

Service Bus provides multiple ways for clients to authenticate themselves with Service Bus. This sample demonstrates the following ways for HTTP/REST clients to authenticate themselves:

  • ACS tokens (Azure Service Bus only)
  • Shared secret tokens (Azure Service Bus only)
  • Shared Access Signature (SAS) tokens (Azure Service Bus and Service Bus for Windows Server 1.1)
  • OAuth tokens / Windows Authentication (Service Bus for Windows Server 1.0 and 1.1)

 

This sample implements a client that obtains a token and then sends a message to and receive the message from Service Bus via HTTP. It performs the following tasks:

  1. Obtain a token.
  2. Creating a queue if it does not exist.
  3. Sending a message m1 with standard and custom properties.
  4. Receiving and processing the message m1.
  5. Delete queue.

 

ACS Token

The client obtains an ACS token by issuing a web request to ACS. The web request contains the namespace owner name and key. ACS will respond with an ACS token that the client copies into the authorization header of its request to Service Bus. This approach is suitable for Azure Service Bus.

Shared Secret Token

The client generates a shared secret token and passes this token to ACS. ACS will respond with an ACS token that the client copies into the authorization header of its request to Service Bus. This approach is suitable for Azure Service Bus.

Shared Access Signature Token

The client generates a shared access (SAS) token that contains a secret namespace key. The client then copies this token into the authorization header of its request to Service Bus. There are two ways to generate a SAS token. The first approach, which is implemented in the sample method GetSasToken1(), uses the Service Bus SharedAccessSignatureTokenProvider. The second approach, which is implemented in the sample method GetSasToken2(), manually constructs this token.

SAS tokens are suitable for Azure Service Bus and Service Bus for Windows Server 1.1.

OAuth Token (Windows Authentication)

The client sends a token request to Service Bus server. The request contains the Windows username and password. Service Bus responds with an OAuth token. The client then copies this token into the authorization header of its request to Service Bus.
Windows Authentication is supported by Service Bus for Windows Server 1.0 and 1.1.

csharp 
const string ClientPasswordFormat = "grant_type=authorization_code&client_id={0}&client_secret={1}&scope={2}";
string requestUri = stsAddress + "/$STS/OAuth/";
string requestContent = string.Format(CultureInfo.InvariantCulture, ClientPasswordFormat, HttpUtility.UrlEncode(userName), HttpUtility.UrlEncode(userPassword), HttpUtility.UrlEncode(scope));
byte[] body = Encoding.UTF8.GetBytes(requestContent);

HttpWebRequest request = WebRequest.Create(requestUri) as HttpWebRequest;
request.AllowAutoRedirect = true;
request.MaximumAutomaticRedirections = 1;
request.Method = "POST";
request.ContentType = "application/x-www-form-urlencoded";
request.ContentLength = body.Length;
request.Timeout = Convert.ToInt32(60*1000, CultureInfo.InvariantCulture);

using (Stream requestStream = request.GetRequestStream())
{
    requestStream.Write(body, 0, body.Length);
}

string rawAccessToken = null;
using (var response = request.GetResponse() as HttpWebResponse)
{
    using (Stream stream = response.GetResponseStream())
    {
        using (var reader = new StreamReader(stream, Encoding.UTF8))
        {
            rawAccessToken = reader.ReadToEnd();
         }
     }
}

string token = string.Format(CultureInfo.InvariantCulture, "WRAP access_token=\"{0}\"", rawAccessToken);

Source Code Files

  • Program.cs – Implements a client that sends and receives messages. Implements various methods to obtain tokens.
  • HttpHelper.cs – Implements helper routines that generate HTTP web requests and process HTTP web responses.
  • BrokerProperties – Implements the standard properties of a Service Bus brokered message.
  • ServiceBusHttpMessage – Implements a message class.
